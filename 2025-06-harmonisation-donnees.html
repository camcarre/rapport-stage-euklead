Harmonisation des données d'achats

Développement d'un outil VBA pour automatiser le nettoyage et l'harmonisation des données d'achats indirects.

Intégration d'un algorithme de similarité Levenshtein pour détecter et corriger les doublons dans les factures et contrats.

Résultat

  • Données d'achats plus fiables
  • Gain de temps significatif pour les analyses de coûts

Détails techniques

Nettoyage et normalisation

  • Uniformisation des libellés fournisseurs (majuscules, accents, trim)
  • Standardisation des formats (dates, montants, TVA)
  • Tables de correspondance pour alias connus (ex: "MICROSOFT FR" → "Microsoft")

Détection de doublons (Levenshtein)

Calcul d'une distance d'édition sur les paires nom + SIRET + code postal avec un seuil de similarité ajustable (ex: 0.85).

  • Couplage blocage: même code postal pour réduire la complexité
  • Priorité aux correspondances exactes SIRET avant fuzzy-matching
  • Journal des fusions généré pour audit

Extrait VBA (pseudo-code)

' Pour chaque fournisseur de référence
For Each ref In fournisseurs
  For Each cand In candidatsDansMemeCP(ref)
    score = Levenshtein(ref.Nom, cand.Nom)
    If cand.SIRET = ref.SIRET Or score > 0.85 Then Fusionner ref, cand
  Next cand
Next ref

Limites & prochaines étapes

  • Cas ambigus nécessitant une validation manuelle
  • Enrichissement futur par scores phonétiques (Soundex/Metaphone)
  • Export des fusions vers Power BI pour contrôle métier
